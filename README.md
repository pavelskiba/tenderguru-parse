# Поиск тендеров (TenderGuru API)

Веб-приложение для поиска тендеров через API TenderGuru. Node.js + Express
отдаёт статический фронтенд (обычный HTML/CSS/JS) и проксирует запросы к
`https://www.tenderguru.ru/api2.3/export`, скрывая ключи API от клиента.

## Параметры API — статус проверки

Схема запроса подтверждена официальной документацией TenderGuru
(раздел «Список закупок (Россия)») и прямыми тестовыми запросами к API
с реальными ключами:

| Фильтр                     | Параметр TenderGuru                          | Статус |
|-----------------------------|-----------------------------------------------|--------|
| Ключевые слова               | `kwords`                                      | ✅ подтверждено |
| Формат ответа                | `dtype=json`                                  | ✅ подтверждено |
| Токен / refresh               | `api_code`, `refresh_key`                     | ✅ подтверждено |
| Закон (44/223/коммерческие)   | `f` = `44` / `223` / `kom`                    | ✅ подтверждено документацией |
| Регион                        | `r{ID}=1` (ID из справочника `config/regions.js`) | ✅ подтверждено (справочник получен через `mode=regions`) |
| Раздел                        | `c{ID}=1` (ID из справочника `config/sections.js`) | ✅ подтверждено (справочник `mode=cat`) |
| Площадка (ЭТП)                | `e{ID}=1` (ID из справочника `config/etp.js`)  | ✅ подтверждено (справочник `mode=eauc`, 100+ площадок, в проекте — отобранные крупные) |
| Цена от/до                    | `price1` / `price2`                           | ✅ подтверждено документацией |
| Дата размещения от/до         | `date_start` / `date_end`, формат `дд.мм.гггг` | ✅ подтверждено документацией |
| Пагинация                     | `page` (платная функция тарифа)               | ✅ подтверждено (размер страницы TenderGuru выбирает сам — не управляется отдельным параметром) |
| ОКПД2                         | `d1`/`d2`/`d3`/`d4` (сегменты кода через точку), только на `/api2.3/export/products` | ⚠️ подтверждено документацией, но не тестировано боевым запросом |
| Область поиска «в документации» | эндпоинт `/api2.3/export/documentation`     | ⚠️ подтверждено документацией, не тестировано боевым запросом |
| Область поиска «в продукции»    | эндпоинт `/api2.3/export/products`          | ⚠️ подтверждено документацией, не тестировано боевым запросом |
| Область поиска «в заголовке»    | отдельного режима в документации нет — используется обычный поиск | ⚠️ приближение |
| Тип заказчика «Государственный» | нет отдельного параметра — если закон не выбран явно, подставляется `f=44,223` | ⚠️ эвристика, не из документации |
| Несколько значений `f` одновременно | `f=44,223` (через запятую)              | ⚠️ предположение, не тестировано |

Если что-то из помеченного ⚠️ не сработает как ожидается — правки нужны
только в `config/params.js` и `lib/tenderguru.js` (функции `buildQuery`,
`resolveEndpoint`, `buildLawParam`), фронтенд трогать не придётся.

### Важное про ответ API

- Ответ — JSON-массив. Первый элемент вида `{"Total": "N"}` — общее число
  найденного по фильтру, не тендер. Остальные элементы — сами тендеры.
  Это уже обрабатывается в `lib/normalize.js`.
- На части тарифов поля `Customer`, `Etp`, `TenderLink` и `TenderNumOuter`
  приходят как текст-заглушка «Только для подписчиков портала: …» — это
  ограничение тарифа TenderGuru, а не ошибка приложения. Ссылка на источник
  поэтому берётся из `TenderLinkInner` (доступна на любом тарифе, ведёт на
  карточку тендера на tenderguru.ru).
- Реальные названия полей ответа: `TenderName`, `Customer`, `Category`,
  `Region`, `Price`, `EndTime`, `Fz`, `TenderLinkInner`, `TenderNumOuter`, `ID`.

## Стек

- Backend: Node.js + Express
- Frontend: статический HTML/CSS/JS в `public/` (без сборки)
- Один сервис, один `package.json` — годится для деплоя на Railway как
  единое приложение.

## Локальный запуск

```bash
npm install
cp .env.example .env
# впишите в .env свои TENDERGURU_API_KEY и TENDERGURU_REFRESH_KEY
npm start
```

Приложение слушает `process.env.PORT` (по умолчанию `3000`), откройте
http://localhost:3000.

## Переменные окружения

| Переменная                | Назначение                                   |
|----------------------------|-----------------------------------------------|
| `TENDERGURU_API_KEY`       | `api_code` — ключ доступа к API TenderGuru    |
| `TENDERGURU_REFRESH_KEY`   | `refresh_key` — код обновления/продления ключа |
| `PORT`                     | Порт сервера (на Railway задаётся автоматически) |

Ключи **никогда** не хардкодятся в коде и не уходят на клиент — они
используются только в `lib/tenderguru.js` на сервере при формировании
запроса к TenderGuru.

## Деплой на Railway

1. Запушьте проект в GitHub-репозиторий (файл `.env` не попадёт в git —
   он в `.gitignore`; `.env.example` используется только как шаблон).
2. В Railway создайте новый проект → Deploy from GitHub repo.
3. Railway определит Node.js-проект (есть `package.json` со скриптом
   `start`, `Procfile` и `railway.json` для Nixpacks) и соберёт его
   автоматически.
4. В настройках проекта на Railway откройте **Variables** и добавьте:
   - `TENDERGURU_API_KEY` = ваш реальный api_code
   - `TENDERGURU_REFRESH_KEY` = ваш реальный refresh_key

   Значения вписываются только в панели Railway — не в код и не в git.
5. Railway сам передаёт приложению переменную `PORT`; сервер уже слушает
   её (`server.js`).
6. После деплоя откройте выданный Railway URL — форма фильтров и поиск
   тендеров должны работать.

## Структура проекта

```
server.js              # Express-сервер, роуты /api/search и /api/dictionaries
lib/tenderguru.js       # Построение запроса к TenderGuru, выбор эндпоинта, ошибки
lib/normalize.js        # Приведение ответа TenderGuru к единому формату
config/params.js        # Соответствие фильтров и параметров TenderGuru API
config/regions.js       # Справочник регионов (ID + название), mode=regions
config/sections.js      # Справочник разделов (ID + название), mode=cat
config/etp.js           # Отобранные крупные площадки (ID + название), mode=eauc
public/                 # Статический фронтенд (HTML/CSS/JS)
```

## API

### `GET /api/dictionaries`
Возвращает списки регионов, разделов и ЭТП (каждый элемент — `{id, name}`)
для построения формы.

### `POST /api/search`
Тело запроса (все поля не обязательны, кроме постраничной навигации):

```json
{
  "kwords": "ремонт кровли",
  "kwordsWhere": "title",
  "okpd": "41.20",
  "laws": ["44fz", "223fz"],
  "customerType": "state",
  "etp": "16",
  "priceFrom": "100000",
  "priceTo": "5000000",
  "dateFrom": "01.01.2026",
  "dateTo": "31.12.2026",
  "regions": ["50", "77"],
  "sections": ["1"],
  "page": 1
}
```

`etp`, `regions`, `sections` — числовые ID из соответствующих справочников
(`config/etp.js`, `config/regions.js`, `config/sections.js`), а не названия.

Ответ:

```json
{
  "ok": true,
  "items": [
    {
      "title": "...",
      "number": "...",
      "customer": "...",
      "region": "...",
      "price": "...",
      "deadline": "...",
      "law": "...",
      "url": "..."
    }
  ],
  "page": 1,
  "onPage": 10,
  "total": 137,
  "hasMore": true
}
```

При ошибке: `{ "ok": false, "code": "...", "message": "..." }` —
обрабатываются неверный ключ, ошибки/лимиты API и пустая выдача.

## Лимиты API

Тариф TenderGuru ограничивает число запросов в месяц (см. раздел
«Стоимость» в документации API — от 30 000/мес на тарифе «Стандарт» и
выше). Каждый поиск на фронтенде — один запрос к API; учитывайте это при
частом использовании и пагинации.
